    - `PostgresRunQueue` uses `JdbcTemplate` and the `run_queue` table. `enqueue` inserts `run_id`, `session_id`, `tenant_id`, `user_id`, `workspace_id`, `trace_id`, `correlation_id`, `input_type`, `input jsonb`, `status='QUEUED'`, `available_at`, `lease_owner=null`, `lease_until=null`, `attempt_count`, `created_at`, and `updated_at`.
    - `claimNext(String workerId, Instant now)` uses a CTE with exact SQL fragment `FOR UPDATE SKIP LOCKED`, claims only `status = 'QUEUED' AND available_at <= ?`, sets `status='CLAIMED'`, `lease_owner`, `lease_until = now + 30 seconds`, increments `attempt_count`, and returns `Optional<QueuedRun>`.
    - `markRunning` updates only `CLAIMED` rows to `RUNNING`.
    - `markTerminal` updates rows not already in terminal statuses to `COMPLETED`, `FAILED`, `CANCELLED`, or `TIMED_OUT`, clears lease fields, and returns true only on transition.
    - `cancelQueuedAndReturn(String runId, String reason, Instant cancelledAt)` atomically updates only `QUEUED` or `CLAIMED` rows to `CANCELLED`, stores `cancel_reason`, clears lease fields, and returns `Optional<QueuedRun>` populated from the row values before/while transitioning. Return `Optional.empty()` if the row is running, terminal, missing, or already cancelled. Do not implement a boolean-only `cancelQueued` method.
    - `removeIfTerminal` deletes only terminal rows and returns true if removed.
    - `InMemoryCancellationRegistry` stores `CancellationToken` per run in `ConcurrentHashMap<String,CancellationToken>`; `tokenFor` creates/registers, `activeToken` returns optional, `requestCancellation` calls token cancellation if present and returns false when no active token exists, and `remove` always removes.
