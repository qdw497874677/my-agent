    - Queue state stays internal; public run status remains separate.
  </behavior>
  <action>
    Implement `PostgresRunQueue` using `JdbcTemplate` and SQL constants.
    - `enqueue(QueuedRun run)` inserts `run_id`, `status='QUEUED'`, `available_at`, `lease_owner=null`, `lease_until=null`, `attempt_count`, `created_at`, and `updated_at`.
    - `claimNext(String workerId, Instant now)` uses a CTE selecting `status = 'QUEUED' AND available_at <= ? ORDER BY available_at, created_at FOR UPDATE SKIP LOCKED LIMIT 1`, updates the row to `status='CLAIMED'`, sets `lease_owner`, `lease_until = now + 30 seconds`, increments `attempt_count`, and returns `Optional<QueuedRun>`.
    - `markRunning` updates `status='RUNNING'`.
    - `markTerminal` updates `status` to the provided terminal status string and clears lease fields.
    - `cancelQueued` updates only rows with `status in ('QUEUED','CLAIMED')` to `status='CANCELLED'` and returns true if one row changed.
